What is the efficient asymptotic running time to find the median of a two-sorted array after merging it??

O(n)

O(log n)

Correct answer

O(n^2)

O(1)

Solution

Hide

Use the divide and conquer paradigm. Compute medians of both the arrays and compare them. Consider m1 be the median of the 1st array a1[] and m2 be the median of the 2nd array a2[]. 

If m1 is greater than m2 then the median may be present in the subarray from the first element of a1[] to m1 or it may be present in the subarray from m2 to the last element of a2[].

If m2 is greater than m1 then the median may be present in the subarray from the first element of a2[] to m2 or it may be present in the subarray from m1 to the last element of a1[].

Repeat above process recursively. Hence, it will take O(log n) time.

A full binary tree (sometimes referred to as a proper or plain binary tree) is a tree in which every node in the tree has either 0 or 2 children.

A perfect binary tree is a binary tree in which all interior nodes have two children and all leaves have the same depth or same level.

In a complete binary tree every level, except possibly the last, is completely filled, and all nodes in the last level are as far left as possible. It can have between 1 and 2h-1 node at the last level h. An alternative definition is a perfect tree whose rightmost leaves (perhaps all) have been removed. Some authors use the term complete to refer instead to a perfect binary tree as defined above, in which case they call this type of tree an almost complete binary tree or nearly complete binary tree. A complete binary tree can be efficiently represented using an array.

In a depth-rst search of an undirected graph G, every edge of G is either a tree edge or a back edge

Correct answer

Solution

Hide

Option 1: TRUE

Depth search explores edge (u,v), it is in the direction from u to v, then v is undiscovered until that time, for otherwise the search would have explored this edge already in the direction from v to u. Thus, (u,v) becomes a tree edge. If the search explores (u,v) rst in the direction from v to u, then (u,v) is a back edge.

Option 2: TRUE

An undirected graph may entail some ambiguity in classification of edges, since (u,v) and (v,u) are really the same edge. Hence, forward and cross edges never occur in a depth-rst search of an undirected graph.

Pre-order traversal in a binary tree is similar to depth first traversal

Option 4: TRUE: Suppose that a depth-rst search produces a back edge (u,v) Then vertex v is an ancestor of vertex u in the depth-rst forest. Thus, G contains a path from v to u, and the back edge (u,v) completes a cycle.
