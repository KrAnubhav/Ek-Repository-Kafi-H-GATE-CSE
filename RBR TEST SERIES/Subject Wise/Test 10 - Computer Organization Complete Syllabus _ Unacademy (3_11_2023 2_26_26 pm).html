If the associativity of a processor cache is doubled while keeping the capacity and block size unchanged, which one of the following is guaranteed to be affected?

Width of tag comparator

Width of set index decoder

Width of way selection multiplexer

All of the above

Correct answer

Solution

Hide

As the number of sets gets decreased, the number of possible cache block entries that a set maps to gets increased. So, we need more tag bits to identify the correct entry. Hence the width of the tag bit comparator increases.

If associativity is doubled, keeping the capacity and block size constant, then the number of sets gets halved. So, the width of the set index decoder can surely decrease.

Width of the way-selection multiplexer must be increased as we have to double the ways to choose from.

Hence option D is the correct answer.

Types of Cache Memory

Given a 4-way set associative cache memory, the main memory size is 8GB and the Tag field has 12 bits. What will be the cache size (consider, memory is byte addressable)

8 MB

Correct answer

128 MB

64 MB

256MB

Solution

Hide

Main memory size = 8GB = 2^33 Bytes (Byte addressable) 


Size of physical address is = 33 bits


Let x be the set offset, y be the word offset.


No. Of sets = 2^x


No. of words in block = 2^y 


1 set = 4 blocks ( 4 way set associative ) 


2^x set contains = 2^x * 4 blocks = 2^(x+2) blocks


Cache size = No of blocks * No of words per blocks 


Cache size = 2^(x+2) * 2^y = 2^(x+y+2) 


Tag bits + x + y = 33 bits 


x + y = 33-12 = 21 bits

Cache size = 2^(21+2) = 2^(23) B = 8 MB

Hence 8 MB is the answer.

Solution

This machine uses 1-address and 2-address instructions.
